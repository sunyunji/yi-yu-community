欢迎你成为易宇社区的贡献者！本指南说明如何以最有效的方式参与项目。

## 贡献工作流程

### 第一步：选择或创建任务

**方式 A：认领已有任务**
1. 浏览 `RESEARCH_DIRECTIONS.md`
2. 找到你感兴趣的任务
3. 在 GitHub Issues 中留言认领

**方式 B：提议新任务**
1. 创建新 Issue，选择"研究提案"或"参与意愿"模板
2. 详细说明你的想法
3. 等待社区反馈

### 第二步：讨论与规划

1. **在 Issue 中沟通**：
   - 明确目标与交付物
   - 讨论技术方案
   - 定义时间表

2. **获取指导**（如需要）：
   - 提出技术问题
   - 请求资源或数据
   - 寻求反馈

### 第三步：开发与实现

1. **Fork 仓库**（如果是代码贡献）
   ```bash
   git clone https://github.com/[你的用户名]/yi-yu-community.git
   cd yi-yu-community
   git checkout -b feature/你的功能名
   ```

2. **遵循代码规范**（见下文）

3. **定期提交**（好的提交信息很重要）
   ```bash
   git commit -m "[feature] 简要描述改动
   
   详细说明（可选）
   "
   ```

4. **保持与主分支同步**
   ```bash
   git fetch origin
   git rebase origin/main
   ```

### 第四步：提交 PR

1. **推送到你的 fork**
   ```bash
   git push origin feature/你的功能名
   ```

2. **创建 Pull Request**
   - 标题清晰（如 `[Feature] 周期识别算法实现`）
   - 描述改动与关联的 Issue
   - 检查 CI 测试是否通过

3. **等待审查与反馈**
   - 核心维护者会审查
   - 可能需要修改
   - 批准后合并

### 第五步：总结与推广

1. **更新文档**（如需要）
2. **发布研究成果**（如适用）
3. **在社区分享**
4. **获得认可** → 添加到 `CONTRIBUTORS.md`

## 测试与验证

### 对于代码

1. **编写单元测试**
   ```python
   def test_period_calculation():
       data = np.array([1, 2, 1, 2, 1, 2])
       result = calculate_period(data)
       assert result["period"] == 2
   ```

2. **运行测试**
   ```bash
   pytest tests/
   ```

### 对于研究

1. **数据验证**：确保数据质量与完整性
2. **结果可重现**：提供代码与数据，使他人能复现
3. **对标测试**：与现有方法比较性能

## 提交信息规范

使用以下前缀确保清晰：

```
[feature]: 新功能或增强（如 [feature] 实现周期识别算法）
[fix]: 修复 bug（如 [fix] 修复白皮书链接错误）
[docs]: 文档更新（如 [docs] 增加快速入门指南）
[refactor]: 代码重构（如 [refactor] 优化预测模型）
[test]: 添加或修改测试（如 [test] 添加单元测试）
[chore]: 其他（如 [chore] 更新依赖）
```

示例：
```
[feature] 实现 AI 周期识别算法

- 基于 FFT 的快速周期检测
- 支持多尺度周期识别
- 包含 95% 测试覆盖率
- 在易宇社区数据集上验证

Closes #123
```

## 文献与参考

### 引用格式

在文档中引用时，使用统一格式：

```markdown
根据《易宇观公理体系：东方科学范式的数学根基》[1]，...

[1] 易宇社区. "易宇观公理体系：东方科学范式的数学根基", 2023.
```

### 学术论文

如果你的贡献基于论文，请：
1. 在文档中明确引用
2. 在代码注释中标注算法出处
3. 在 `docs/REFERENCES.md` 中补充

## 审查流程

### 什么时候你的 PR 会被审查？

1. **自动检查通过**（代码格式、测试等）
2. **至少一位核心贡献者审查**
3. **讨论与改进**（如需要）
4. **批准与合并**

### 如何处理反馈？

1. **认真阅读注释**
2. **在本地修改**
3. **再次提交**
4. **继续对话直到解决**

## 常见问题

**Q: 我完全是初学者，能参与吗？**
A: 完全可以！从翻译、文档整理、数据收集等开始。

**Q: 我没有写过开源代码，怎么办？**
A: 这是学习的好机会。我们会帮助指导。

**Q: 我的项目失败了，会怎样？**
A: 没关系。失败也是学习。分享你的发现与教训。

**Q: 代码或论文获得认可有什么奖励吗？**
A: 主要奖励是：
- 成为开源社区的一部分
- 参与推动新范式的建设
- 获得学术认可与名誉
